# BMS Manager

iOS app (SwiftUI + CoreBluetooth + SwiftData) for monitoring lithium batteries that expose a Bluetooth BMS — Impulse Lithium, Renogy, LiTime, Eco-Worthy, etc., most of which speak the **JBD/Xiaoxiang** BLE protocol.

## Features

- Scan for nearby BMS-equipped batteries
- Save batteries to a dashboard with custom names
- Group batteries in **series** or **parallel** with aggregate stats
- Per-battery live stats: voltage, current, SOC %, cycles, temperature
- Time-to-empty (when discharging) and time-to-full (when charging)

## Requirements

- Xcode 16+
- iOS 17+ deployment target (uses SwiftData and modern SwiftUI)
- A physical iPhone/iPad — Bluetooth doesn't work in the simulator

## Generating the Xcode project

This repo ships sources only. Generate the `.xcodeproj` with [XcodeGen](https://github.com/yonsm/XcodeGen):

```bash
brew install xcodegen
cd /Users/ben/projects/bms-manager
xcodegen generate
open BMSManager.xcodeproj
```

Or, if you'd rather not use XcodeGen: in Xcode, **File → New → Project → iOS App** (SwiftUI, Swift, "BMSManager"), then drag the `Sources/` folder into the project navigator and replace the autogenerated `Info.plist` keys with the ones in `Sources/Info.plist`.

## On a real device

1. Plug in your iPhone, select it as the run target
2. In **Signing & Capabilities**, set your team
3. ⌘R — first launch will prompt for Bluetooth permission

## Protocol notes

The JBD basic-info request is hardcoded in `Sources/Bluetooth/JBDProtocol.swift`. If your BMS uses a different protocol (Daly, Seplos, ANT), swap that file. Service `FF00` / notify `FF01` / write `FF02` are the de-facto BLE BMS UUIDs.
